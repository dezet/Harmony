# Harmony Frontend (React) — Agent Guide

This is the React SPA for Harmony. The backend is Elixir/Phoenix in the parent directory
(`../lib`, `../test`). The SPA talks to it over a JSON API (`/api/v1/*`) and a Phoenix Channel
(`/socket`).

## Stack (as installed)

- **TypeScript + Vite** — root: `elixir/assets`, builds to `elixir/priv/static/app`.
- **React 19** + **React Router v7** (routes in `src/App.tsx`).
- **React Query (TanStack Query v5)** — the single client-side source of truth for server data.
- **Phoenix Channels** via the `phoenix` npm client — live dashboard data hydrates the React Query
  cache (see `src/lib/socket.ts`).
- **React Hook Form + Yup** for forms.
- **shadcn/ui** with the **default `base-nova` style** (built on **Base UI**, not Radix) +
  **Tailwind v4**. `components.json` records the config.
- **Vitest + React Testing Library** for tests.

## Conventions

- `src/components/ui/*` are generated by the shadcn CLI — **do not hand-edit them**.
- **Before writing a custom component, check shadcn first:** `npx shadcn@latest add <name>`.
- Lean on the **default shadcn (base-nova) theme**. Avoid ad-hoc CSS; use Tailwind utilities and the
  theme tokens defined in `src/index.css`.
- **No `fetch` in components.** All HTTP goes through `src/lib/api.ts` + React Query hooks. Live
  data arrives through the channel hook in `src/lib/socket.ts`.
- The wire contract (server payload shapes) lives in `src/types/contract.ts`. Keep it in sync with
  `SymphonyElixirWeb.Presenter` in the Elixir app.
- Feature code lives under `src/features/<feature>/` or `src/routes/`. Files that change together
  live together.

## Path alias

`@/*` → `src/*`. Configured in `vite.config.ts` (resolve.alias) and `tsconfig.app.json` (paths),
plus `tsconfig.json` (for the shadcn CLI's resolver). If `npx shadcn add` ever writes into a literal
`@/` directory, move the files into `src/` and re-check the alias config.

## Run

- **Dev:** start Phoenix (the OTP app boots the server), then `npm run dev` here. Vite proxies
  `/api` and `/socket` to Phoenix on `http://localhost:${HARMONY_PORT:-4000}`. Open
  http://localhost:5173/. If Phoenix runs on another port, use `HARMONY_PORT=<port> npm run dev`.
- **Tests:** `npm run test -- --run`
- **Typecheck:** `npm run typecheck`
- **Build:** from `elixir/`, `mix assets.build` (or `npm run build` here).
- **Browser E2E:** from `elixir/`, `make e2e` runs the deterministic React SPA Playwright harness.

## Routing note

The Phase 3 cutover is complete: Vite builds with `base: "/"`, Phoenix serves `priv/static/app`
from `/`, and React Router owns `/` (Overview), `/runtime`, `/projects`, `/projects/new`, and
`/projects/:id/edit`. The shell is a project sidebar (`src/components/layout/Sidebar.tsx`) +
breadcrumb header; page features live under `src/features/{overview,runtime,projects}/`.
